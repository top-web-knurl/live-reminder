# План: Оптимизация первичной загрузки приложения

## Проблема
При запуске Electron-приложения долгая первичная загрузка из-за блокирующих `require()` нативных модулей (`canvas`, `sqlite3`) и лишних операций при старте.

## Найденные узкие места

| # | Проблема | Файл | Вклад в задержку |
|---|----------|------|------------------|
| 1 | `require('canvas')` загружается синхронно, но используется только в `updateTrayBadge()` | main.js:4 | ~5-15ms |
| 2 | `require('sqlite3')` загружается через цепочку импортов, хотя БД открывается лениво | src/db.js:1 | ~10-30ms |
| 3 | `fs.readFileSync(trayIconPath)` читает иконку при каждом обновлении бейджа | main.js:312 | I/O на каждый вызов |
| 4 | Миграции БД (`PRAGMA table_info` + 7× `ALTER TABLE`) запускаются при каждом старте | src/db.js:39-63 | Лишние запросы |
| 5 | `updateTrayBadge()` вызывается при старте даже если 0 непросмотренных | main.js:458 | Лишний DB + canvas |

## Решения

### 1. Ленивая загрузка `canvas` (main.js)
- Убрать `const { createCanvas, loadImage } = require('canvas')` из строки 4
- Добавить `require('canvas')` внутри `updateTrayBadge()` при первом вызове
- Кэшировать модуль в переменную

### 2. Ленивая загрузка `sqlite3` (src/db.js)
- Убрать `const sqlite3 = require('sqlite3').verbose()` из строки 1
- Добавить `require('sqlite3').verbose()` внутри `initDB()` при первом вызове
- Кэшировать модуль в переменную

### 3. Кэширование иконки трея (main.js)
- При первом чтении `icon.png` сохранить буфер в переменную
- При последующих вызовах `updateTrayBadge()` использовать кэш
- Инвалидировать кэш только если файл изменился (по mtime)

### 4. Оптимизация миграций (src/db.js)
- Использовать `PRAGMA user_version` для проверки версии схемы
- Текущая версия схемы = 2 (reminders + templates)
- При старте: если `user_version < 2` — выполнять миграции и установить `user_version = 2`
- При `user_version = 2` — пропускать все `PRAGMA table_info` и `ALTER TABLE`

### 5. Отложить `updateTrayBadge()` (main.js)
- Не вызывать `updateTrayBadge()` при старте
- Вызвать его после первой успешной загрузки данных в renderer (через IPC-событие или по таймеру 500ms)

## Файлы для изменения

- `main.js` — ленивый canvas, кэш иконки, отложенный badge
- `src/db.js` — ленивый sqlite3, миграции через user_version

1. `npm start` — время до показа окна должно уменьшиться
2. Проверить что треей с бейджем работает
3. Проверить что миграции не ломаются (удалить БД, перезапустить)
4. Проверить что экспорт/импорт работают
